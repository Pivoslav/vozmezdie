Design Pitch & Grand Plan

Expert-grounded LLM evaluation on Cold War archival documents

Feb 2025

Vision & Goals

What it is: Compare human-coded ground truth to LLM extractions — content categories and framing strategies per segment.

  • Review — Spot patterns, mismatches, model errors
  • Assess — Blind or standard manual labelling
  • Reference — Taxonomy definitions, terms in context

Principles: Focus (one task at a time) | Bilingual parity | Low friction (single HTML) | Accessibility

User Personas

Reviewer — Compare LLM vs human; write findings. Pain: switching table/text; finding mismatches; getting lost in long docs.
Assessor — Label segments; stay blind or compare. Pain: taxonomy lookup; chunk progress; export/import.
Researcher — Cross-doc patterns; term frequency. Pain: no aggregate view; glossary separate from data.

Primary Flows to Optimize

  1. "Show me mismatches" — Filter to mismatches; jump table to text
  2. "What does this term mean?" — Hover/click -> definition tooltip or glossary
  3. "Where am I?" — Scroll sync; table row highlights
  4. "Compare across documents" — Summary dashboard; per-doc stats

Demo: Scroll Sync Concept

When you scroll one panel, the other follows. Keeps Eng/Rus aligned for bilingual comparison.

// Pseudocode: sync scroll between two divs
panels.forEach(p => p.addEventListener('scroll', () => {
  if (syncing) return;
  syncing = true;
  otherPanel.scrollTop = p.scrollTop;
  otherPanel.scrollLeft = p.scrollLeft;
  requestAnimationFrame(() => syncing = false);
}));

Implement with scrollTop / scrollLeft sync; debounce or use passive listeners for perf.

Demo: Click Span -> Table Row

Each span in the text view has data-row-index. On click:

span.addEventListener('click', () => {
  const row = table.querySelector(`tr:nth-child(${index + 1})`);
  row.scrollIntoView({ behavior: 'smooth', block: 'center' });
  row.classList.add('table-row-highlight');
  setTimeout(() => row.classList.remove('table-row-highlight'), 2000);
});

Reverse: click table row -> scroll text view to that segment.

Sepia, serif. Document-analysis feel.

Monochrome + data colour only.

Optional; long sessions.

Technical Path

Current: Single HTML; inline CSS/JS in Python.

Recommendation: Progressive enhancement. Add features with vanilla JS. No framework.

When report module exceeds ~1200 lines: split to template (Jinja or similar). Python builds data; template renders.

Avoid SPA unless we need real-time updates.
