# Experiment: LLM Assessor Without Generic / Neutral Framing

**For a Cursor agent.** Re-run the assessment experiment with framing only, excluding "Generic / Neutral Language". Output goes to new files; nothing is overwritten.

---

## 1. Goal

Compare agent-assessed framing to ground truth, but **without** the "Generic / Neutral Language" framing option. The agent must choose one of the remaining four framing strategies for every segment. This tests whether forcing a non-generic choice changes agreement patterns.

---

## 2. Framing Strategies (Use Only These 4)

| Framing strategy |
|------------------|
| Institutional / Bureaucratic Lingo |
| Ideological Framing (Discrediting) |
| Ideological Phrasing (Normalizing) |
| Action-Focused Language |

**Do not use:** Generic / Neutral Language, Generic / Neutral

For segments that would otherwise be "Generic / Neutral", choose the closest of the four:
- Neutral bureaucratic language → Institutional / Bureaucratic Lingo
- Neutral factual reporting → Institutional / Bureaucratic Lingo or Action-Focused Language
- When genuinely ambiguous, prefer Institutional / Bureaucratic Lingo (default for formal archival tone)

---

## 3. Content Categories (Unchanged)

Use the full taxonomy: Actions | Actors | Places | Time | Documents | Context and Concepts | Legal Framework

---

## 4. Output Files (Do Not Overwrite)

| File | Purpose |
|------|---------|
| `data/output/agent_assessments_no_generic.json` | Your assessments (same shape as agent_assessments.json) |
| `data/output/comparison_results_no_generic.json` | Comparison vs ground truth (generated by pipeline) |
| `data/output/manual_analysis_report_no_generic.html` | Report for this experiment |

**Never modify:** `agent_assessments.json`, `comparison_results.json`, `manual_analysis_report.html`

---

## 5. Workflow

### Step 1: Export segments (if not already done)

```bash
python scripts/export_segments_filtered.py <doc_id> [doc_id ...]
```

Or for blind (no labels):

```bash
python scripts/export_segments_only.py <doc_id>
```

Output: `data/output/segments_filtered_<doc_id>.json` or `segments_only_<doc_id>.json`

### Step 2: Assess each segment

- Read `data/russian_originals/<doc_id>.txt` (and English if available)
- For each segment: assign **content_category** (full 7) and **framing** (4 only, no Generic)
- Record in a script that writes to `agent_assessments_no_generic.json`

Use `scripts/write_fresh_assessment_*.py` as templates, but:
- Change `ASSESSMENTS_PATH` to `ROOT / "data" / "output" / "agent_assessments_no_generic.json"`
- Use only the 4 framing strategies above

### Step 3: Run pipeline with experiment files

```bash
python run.py --agent-assessments --agent-assessments-file=data/output/agent_assessments_no_generic.json --output-report=data/output/manual_analysis_report_no_generic.html
```

(Run.py supports `--agent-assessments-file` and `--output-report`.)

### Step 4: Compare

Open `manual_analysis_report_no_generic.html` alongside `manual_analysis_report.html` to compare framing agreement, mismatch flow, and term-framing heatmaps.
