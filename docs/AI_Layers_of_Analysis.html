AI Layers of Analysis

The Task

The task was to apply a structured taxonomy to Cold War archival documents and compare AI-assisted or automated classifications against human expert ground truth. Each document is split into segments (phrases or entities), and each segment receives two labels: a content category (what kind of information it is) and a framing strategy (how it is expressed). The taxonomy includes seven content categories: Actions, Actors, Places, Time, Documents, Context and Concepts, Legal Framework. It also includes five framing strategies: Generic/Neutral Language, Institutional/Bureaucratic Lingo, Ideological Framing (Discrediting), Ideological Phrasing (Normalizing), and Action-Focused Language. The goal was to see how well AI or AI-assisted analysis could reproduce expert judgments on both dimensions.

Methods and Approach

We built a modular pipeline with five stages: ingest, extraction, ground truth loading, comparison, and report generation. Documents are ingested from Russian originals, and ground truth comes from human-coded HTML tables with pre-defined segments. The comparison step aligns LLM output to ground truth by matching segment text (exact match first, then containment), and then computes agreement on category, framing, and both.

The primary method was local LLM classification. We exported segments from ground truth (body rows only, so segment boundaries align with the human-coded tables), then ran a local model (GPT-OSS) to classify each segment one-by-one. For each segment, the model receives a prompt that lists the allowed content categories and framing strategies, the segment text in English and Russian, and optional context. The model returns a JSON object with the content category and framing. Classification runs at roughly two to three seconds per segment. Results are merged into the pipeline and compared against ground truth. We focused on per-segment classification (where segments are fixed by ground truth) rather than full-document extraction, because a single LLM call per document produces different segment boundaries and does not align with the human-coded structure.

We chose to use local LLM classification (GPT-OSS) rather than cloud APIs, so we could run the full pipeline on our own hardware and keep segment boundaries aligned with ground truth. We treated Russian as the primary source and used it for alignment when available. We adopted a dual taxonomy (content plus framing) so we could study both what is said and how it is said as separate dimensions. Separating content from framing lets us see where the LLM performs better or worse on each, and the bilingual corpus (Russian originals with English translations) supports research on how LLMs handle multilingual archival material. The model receives both languages; alignment can use either. To align LLM output rows to ground truth rows, we match segment text in two steps: first we pair rows where the normalized text matches exactly, then for any remaining unmatched rows we pair them where one phrase contains the other (e.g., a shortened or paraphrased variant). Segment boundaries are fixed by ground truth, so the LLM does not segment; it only labels. This design keeps the evaluation focused on classification accuracy rather than segmentation.

Choices, Challenges, Successes, and Failures

Several challenges emerged along the way. Alignment proved sensitive to phrasing: if the LLM paraphrases or the segment text differs slightly, matching fails. Framing turned out to be more subjective than content. The same phrase can be read as neutral or institutional depending on context, and some categories (e.g., Legal Framework, Places) appear rarely, so accuracy estimates for them are noisier. The archival and institutional context adds another layer: Cold War KGB documents use specific institutional language and ideological framing. The taxonomy was designed to capture these features, and the work shows how well a general-purpose LLM can recognize them.

Despite these challenges, we achieved a number of successes. Sixteen documents were fully assessed, with roughly 2,900 aligned segments across the corpus. We built a modular pipeline and a reusable evaluation framework. A central deliverable is the interactive report, which we developed as a research tool in its own right. The comparison tables let researchers search, filter by category or framing, and inspect each segment side by side with its LLM and human labels. The document text view presents the full document in English and Russian with synchronized scrolling; segments are highlighted by category and framing, and users can filter by label to focus on specific types of content or disagreement. The glossary provides definitions and examples for each category and framing strategy, with search and links that jump directly to segments in the document. These interactive elements turn the report from a static summary into an explorable interface where researchers can drill into mismatches, trace vocabulary across documents, and build intuition about where the model succeeds or fails.

The report also includes a suite of visualizations. Word clouds, category-by-framing heatmaps, per-document stacked bars, and pie charts show distribution at a glance. A document similarity matrix (cosine similarity of framing profiles) reveals which documents cluster by language style. A document fingerprint view compares framing proportions across the corpus. A places map extracts and geocodes locations from Places-tagged segments, supporting spatial analysis. Segment length vs accuracy charts show whether longer segments are classified more reliably. Voyant tools (Cirrus, Links, Bubblelines, Constellations) are embedded for corpus-level text analysis. Each visualization includes a "How is this calculated?" section so the methodology is transparent. Bilingual support (EN/UK) and glossary definitions drawn from the taxonomy support broader use. Accuracy varies across documents; some documents show higher agreement than others, which may reflect document length, clarity of language, or the distribution of categories and framings in the text. The report surfaces per-document stats so researchers can explore these patterns. The report also extracts terms by category and framing, which supports qualitative exploration of what the model learns: which phrases cluster in Action-Focused vs Institutional, which vocabulary signals Ideological (Discrediting).

We also encountered failures. Full-document LLM extraction produced different segment boundaries than ground truth, so alignment was poor. Both-match accuracy remained low at 46.5%, indicating that getting both category and framing right at once is difficult.

Accuracy

Across 2,913 aligned segments in 16 documents, category accuracy reached 65.8%, framing accuracy 73.2%, and both-match 46.5%.

Main Outcome

Conceptually, we showed that content categorization is more tractable than framing detection. Category is easier to agree on; framing depends on subtle judgments about tone, institutional language, and ideology. The dual taxonomy (content plus framing) provides a structured way to study both dimensions and to identify where the LLM struggles.

Practically, we built a reusable pipeline for expert-grounded LLM evaluation on archival documents. The framework can ingest documents, run local LLM classification (GPT-OSS per-segment), compare results to ground truth, and produce an interactive report. The report is a substantial part of the contribution: the interactive tables and visualizations are designed so that researchers can explore the data rather than only read summary statistics. Filtering, search, bilingual highlighting, and linked glossary entries make it possible to move from a high-level pattern (e.g., low framing accuracy on a given document) to the specific segments that drive it. This supports scaling to more documents, comparison across models or prompt variants, and exploration of agreement patterns and failure modes. The pipeline is modular and config-driven, so researchers can swap models, adjust prompts, or add new documents without changing the core logic. Outputs (comparison JSON, report HTML) can be regenerated from saved results.

As a novel research contribution, we applied AI-assisted analysis to Cold War archival documents using a dual taxonomy of content and framing, and evaluated it against expert ground truth. The work demonstrates how local LLMs can be used to classify archival text along both semantic and rhetorical dimensions, and provides a framework for future work on model comparison, prompt engineering (e.g., few-shot, chain-of-thought), taxonomy refinement (splitting or merging categories), and scaling to more documents or corpora.
